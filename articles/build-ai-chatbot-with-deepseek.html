从零开始：用DeepSeek API构建一个AI聊天机器人（附Python和curl代码）

如果你正在寻找一个强大、便宜（甚至免费）的AI API来构建自己的聊天机器人，DeepSeek绝对值得关注。它不仅在数学、编码等任务上表现出色，而且API定价非常亲民。今天我们就手把手教你用DeepSeek API搭建一个能跑起来的聊天机器人，全程代码可复制。

为什么选择DeepSeek API？

在开始写代码之前，先简单说说DeepSeek的优势：

  • 性能强劲：在多个基准测试中接近GPT-4水平，尤其在中文和编程场景
  • 价格良心：输入0.14元/百万token，输出0.28元/百万token（约是GPT-4的1/20）
  • 上下文长：支持128K上下文窗口，能处理长篇对话或文档
  • 开源友好：模型完全开源，API调用方式兼容OpenAI格式

准备工作：获取API密钥

第一步，你需要一个DeepSeek API密钥。去DeepSeek官网注册账号，在控制台创建API密钥。注意：新用户通常会赠送几百万token的免费额度，足够你测试很久。

拿到密钥后，把它保存到环境变量或配置文件中，千万不要硬编码在代码里。

方法一：用curl快速测试API联通性

在写任何代码之前，先用curl确认API是否正常工作。打开终端，执行以下命令（替换YOUR_API_KEY）：

curl https://api.deepseek.com/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "deepseek-chat",
    "messages": [
      {"role": "system", "content": "你是一个友好的助手。"},
      {"role": "user", "content": "你好，请用一句话介绍自己"}
    ],
    "stream": false
  }'
如果返回类似这样的JSON，说明API连接成功： 
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "你好！我是DeepSeek，一个由深度求索公司创造的AI助手，很高兴为你服务！"
    },
    "finish_reason": "stop"
  }]
}
小技巧：把"stream": true改为true可以实现流式输出，适合需要实时显示文本的场景。

方法二：用Python构建基础聊天机器人

安装依赖

pip install openai  # DeepSeek API兼容OpenAI SDK

最小可用版本

下面的代码实现了一个命令行聊天机器人，支持多轮对话： 
from openai import OpenAI
import os

# 初始化客户端（注意base_url要改成DeepSeek的地址）
client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),  # 从环境变量读取
    base_url="https://api.deepseek.com/v1"
)

def chat_with_deepseek():
    """简单的命令行对话循环"""
    messages = [
        {"role": "system", "content": "你是一个乐于助人的AI助手。请用中文回答。"}
    ]
    
    print("🤖 DeepSeek聊天机器人已启动（输入exit退出）")
    print("-" * 40)
    
    while True:
        user_input = input("\n👤 你: ")
        if user_input.lower() in ["exit", "quit"]:
            print("👋 再见！")
            break
        
        # 添加用户消息
        messages.append({"role": "user", "content": user_input})
        
        try:
            # 调用API
            response = client.chat.completions.create(
                model="deepseek-chat",
                messages=messages,
                temperature=0.7,  # 控制创造性（0-2）
                max_tokens=2048,
                stream=False
            )
            
            # 获取AI回复
            ai_reply = response.choices[0].message.content
            print(f"\n🤖 DeepSeek: {ai_reply}")
            
            # 将AI回复加入历史，支持上下文
            messages.append({"role": "assistant", "content": ai_reply})
            
        except Exception as e:
            print(f"\n❌ 出错了: {str(e)}")

if __name__ == "__main__":
    chat_with_deepseek()
运行方式：在终端执行export DEEPSEEK_API_KEY="你的密钥"，然后运行Python脚本。

进阶：加入流式输出和错误重试

生产环境建议加入流式输出和重试机制，体验会好很多： 
import time
from openai import OpenAI

def stream_chat(client, messages, retries=3):
    """流式输出带重试"""
    for attempt in range(retries):
        try:
            response = client.chat.completions.create(
                model="deepseek-chat",
                messages=messages,
                stream=True,  # 开启流式
                temperature=0.7
            )
            
            full_content = ""
            print("🤖 DeepSeek: ", end="", flush=True)
            
            for chunk in response:
                if chunk.choices[0].delta.content:
                    content = chunk.choices[0].delta.content
                    print(content, end="", flush=True)
                    full_content += content
            
            print()  # 换行
            return full_content
            
        except Exception as e:
            if attempt < retries - 1:
                wait_time = 2 ** attempt  # 指数退避
                print(f"\n⚠️ 请求失败，{wait_time}秒后重试...")
                time.sleep(wait_time)
            else:
                print(f"\n❌ 重试{retries}次后仍失败: {str(e)}")
                return None

方法三：构建带Web界面的聊天机器人

命令行太简陋？我们用Flask快速搭建一个网页版： 
from flask import Flask, request, jsonify
from openai import OpenAI
import os

app = Flask(__name__)
client = OpenAI(
    api_key=os.getenv("DEEPSEEK_API_KEY"),
    base_url="https://api.deepseek.com/v1"
)

# 存储会话（生产环境请用Redis等）
sessions = {}

@app.route("/chat", methods=["POST"])
def chat():
    data = request.json
    session_id = data.get("session_id", "default")
    user_message = data.get("message", "")
    
    # 初始化或获取会话历史
    if session_id not in sessions:
        sessions[session_id] = [
            {"role": "system", "content": "你是一个专业的技术顾问。"}
        ]
    
    sessions[session_id].append({"role": "user", "content": user_message})
    
    try:
        response = client.chat.completions.create(
            model="deepseek-chat",
            messages=sessions[session_id],
            temperature=0.7,
            max_tokens=2048
        )
        
        ai_reply = response.choices[0].message.content
        sessions[session_id].append({"role": "assistant", "content": ai_reply})
        
        return jsonify({"reply": ai_reply, "session_id": session_id})
    
    except Exception as e:
        return jsonify({"error": str(e)}), 500

@app.route("/reset", methods=["POST"])
def reset():
    data = request.json
    session_id = data.get("session_id", "default")
    if session_id in sessions:
        del sessions[session_id]
    return jsonify({"status": "ok"})

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000, debug=True)

常见问题与优化技巧

1. 控制Token消耗

DeepSeek按token计费，建议设置max_tokens限制，避免单次回复过长： 
response = client.chat.completions.create(
    model="deepseek-chat",
    messages=messages,
    max_tokens=1024  # 单次回复最多1024个token
)

2. 系统提示词（System Prompt）的妙用

通过精心设计system prompt，可以大幅提升回答质量： 
system_prompt = """
你是一个资深Python技术顾问。请遵循以下规则：

  • 所有代码示例必须包含注释
    • 

  • 优先推荐标准库解决方案
    • 

  • 如果用户问题涉及第三方库，需说明安装方法
    • 

  • 回答末尾可以给出优化建议
    • 

"""

3. 处理长上下文

DeepSeek支持128K上下文，但history越长费用越高。可以设置最大历史轮数： 
MAX_HISTORY = 20  # 保留最近20轮对话

def trim_history(messages):
    if len(messages) > MAX_HISTORY + 1:  # +1是system消息
        # 保留system消息和最近MAX_HISTORY条
        return [messages[0]] + messages[-(MAX_HISTORY):]
    return messages

4. 错误处理的最佳实践

import logging
from openai import RateLimitError, APITimeoutError

def safe_api_call(client, messages):
    try:
        return client.chat.completions.create(...)
    except RateLimitError:
        logging.warning("触发速率限制，等待1秒")
        time.sleep(1)
        return safe_api_call(client, messages)  # 递归重试
    except APITimeoutError:
        logging.error("API超时")
        return None
    except Exception as e:
        logging.critical(f"未知错误: {e}")
        return None
