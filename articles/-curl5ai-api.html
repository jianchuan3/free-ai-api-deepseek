为什么“免费”不等于“能用”？

    2. 智谱GLM：长上下文之王

    智谱的免费模型glm-4-flash支持128K上下文，这意味着你可以直接把整个文档丢进去让它总结。这对于处理大型代码库或长文档非常有用。 Python调用示例（使用requests库）： 
    import requests
import json

url = "https://open.bigmodel.cn/api/paas/v4/chat/completions"
headers = {
    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"
}
data = {
    "model": "glm-4-flash",
    "messages": [
        {"role": "user", "content": "请解释什么是RAG技术，并给出一个简单的实现思路"}
    ],
    "stream": False
}

response = requests.post(url, headers=headers, data=json.dumps(data))
print(response.json()['choices'][0]['message']['content'])
    实测结果： 响应速度中等（约3-4秒），但对长文本的理解非常出色。如果你需要处理复杂文档，智谱是首选。

    3. 通义千问：简单直接

    二、curl调用大模型的3个实用技巧

    技巧1：如何处理API Key安全？

    直接写死在curl命令里不安全。建议： 
    # 设置环境变量
export DEEPSEEK_API_KEY="sk-your-key"

# 在curl中引用
curl -X POST ... -H "Authorization: Bearer $DEEPSEEK_API_KEY"

    技巧2：流式输出 vs 非流式输出

    当你用curl调用大模型时，默认会等待完整响应。如果你想要实时看到输出（类似ChatGPT打字效果），可以添加"stream": true： 
    curl -X POST ... -d '{"stream": true, ...}' | while read -r line; do
    echo "$line"
done

    技巧3：多轮对话的上下文管理

    大多数免费API不自动保存上下文，你需要手动维护消息列表： 
    # 初始对话
messages = [{"role": "user", "content": "你好"}]
# 获取回复后追加
messages.append({"role": "assistant", "content": response})
messages.append({"role": "user", "content": "刚才的话是什么意思？"})
